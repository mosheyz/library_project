# library_project

ניהול ספריה:
המערכת מאפשרת לנהל את הספרים בספריה ולשנות אותם.
וכן לנהל ולשנות את המשתמשים.
המערכת מנוהלת על ידי שרת api,
וזו הדרך לתקשר איתה בצורת לקוח-שרת.

============================================

פקודות להרצת docker:

docker run
--name mysql-w7
-e MYSQL_ROOT_PASSWORD=root
-p 3306:3306 -d mysql:8

מבנה הטבלאות:

books:

id -> מספר מזהה יחודי לכל ספר
title -> שם הספר | חובה
author -> שם המחבר | חובה
genre -> סוגה - ערכים מסויימים מתוך רשימה מובנית | חובה
is available -> האם הספר זמין להשאלה | חובה
borrowed by member id -> אם הספר בהשאלה - מספר מזהה של השואל


members:

id -> מספר מזהה יחודי לכל משתמש | חובה
name -> שם המשתמש | חובה
email -> אימייל | חובה
is active -> האם המשתמש פעיל | חובה
total borrows -> כמה ספרים המשתמש השאיל עד עכשיו | חובה

============================================

חוקי המערכת:

זאנר -> חייב להיות מתוך מאגר מסויים
אימייל -> חייב להיות ייחודי
לא פעיל -> חבר לא פעיל - לא יכול לשאול ספר
לא זמין -> אי אפשר להשאיל ספר לא זמין
מקסימום -> אי אפשר להשאיל יותר מ3 ספרים
החזרה -> אפשר להחזיר רק אם המשתמש הוא זה שהשאיל את הספר

============================================

רשימת endpoints:


ספרים:

יצירת ספר -> POST /books
רשימת ספרים -> GET /books
ספר לפי id -> GET /books/{id} 
עדכון פרטי ספר -> PUT /books/{id}
עדכון החזרה -> PUT /books/{id}/return/{member_id}
עדכון השאלה -> PUT /books/{id}/borrow/{member_id}
סכום ספרים/
סכום ספרים זמינים/
סכום ספרים לא זמינים -> GET /summary/reports
סכום ספרים לפי זאנר -> GET /reports/books-by-genre
סכום ספרים שמשתמש מחזיק כרגע -> PUT /books/{id}/borrow/{member_id}

משתמשים:

יצירת משתמש -> POST /members
רשימת משתמשים -> GET /members
משתמש לפי id -> GET /members/{id}
עריכת פרטי משתמש -> PUT /members/{id}
עדכון שהמשתמש לא פעיל -> PUT /members/{id}/deactivate
עדכון שמשתמש פעיל -> PUT /members/{id}/activate
הוספת 1 לסכום השאלות של משתמש -> PUT /books/{id}/borrow/{member_id}
סכום משתמשים פעילים -> GET /reports/summary
משתמש עם מקסימום השאלות -> GET /reports/top-member

============================================

זרימת המערכת:

ברגע שמריצים את השרת נוצרות 2 הטבלאות.

המשתמש יוצר משתמש חדש,
המערכת בודקת אם האימייל ייחודי.

המשתמש יוצר ספר,
המערכת בודקת אם הז'אנר חוקי.

המשתמש מבקש לשאול ספר ->
המערכת בודקת האם -
הספר קיים וזמין,
המשתמש קיים ופעיל,
מושאל למשתמש פחות מ3 ספרים כרגע.

במידה וכל הבדיקות הצליחו:
הסטטוס של הספר משתנה ללא-זמין,
מתעדכן בפרטי הספר שהוא מושאל על ידי המשתמש,
מונה ההשאלות עולה ב1.

המשתמש מבקש להחזיר ספר ->
המערכת בודקת האם -
הספר קיים ומושאל לאותו משתמש,
המשתמש קיים.

במידה וכל הבדיקות הצליחו:
הסטטוס של הספר משתנה לזמין,
מתעדכן בפרטי הספר שהוא לא מושאל כרגע.

הוראות הרצה:

הקמת והרצת שרת:

python -m venv venv
.venv\Scripts/activate # הקמת סביבה מבודדת
pip install 'fastapi[standard]' # הורדת הספריות הנצרכות לשרת
pip freeze > requirements.txt # הכנסת ההורדות לקובץ

uvicorn main:app --reload # הרצת הקובץ שבו השרת
http://127.0.0.1:8000 # קישור גישה להרצת הבקשות מהשרת
