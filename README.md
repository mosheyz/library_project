# library_project

### ניהול ספריה:

המערכת מאפשרת לנהל את הספרים בספריה ולשנות אותם.
וכן לנהל ולשנות את המשתמשים.
המערכת מנוהלת על ידי שרת api,
וזו הדרך לתקשר איתה בצורת לקוח-שרת.

### מבנה הטבלאות:

**books:**

id -> מספר מזהה יחודי לכל ספר<br>
title -> שם הספר | חובה<br>
author -> שם המחבר | חובה<br>
genre -> סוגה - ערכים מסויימים מתוך רשימה מובנית | חובה<br>
is available -> האם הספר זמין להשאלה | חובה<br>
borrowed by member id -> אם הספר בהשאלה - מספר מזהה של השואל<br>


**members:**

id -> מספר מזהה יחודי לכל משתמש | חובה<br>
name -> שם המשתמש | חובה<br>
email -> אימייל | חובה<br>
is active -> האם המשתמש פעיל | חובה<br>
total borrows -> כמה ספרים המשתמש השאיל עד עכשיו | חובה<br>

### חוקי המערכת:

זאנר -> חייב להיות מתוך מאגר מסויים
אימייל -> חייב להיות ייחודי
לא פעיל -> חבר לא פעיל - לא יכול לשאול ספר
לא זמין -> אי אפשר להשאיל ספר לא זמין
מקסימום -> אי אפשר להשאיל יותר מ3 ספרים
החזרה -> אפשר להחזיר רק אם המשתמש הוא זה שהשאיל את הספר

---
### רשימת endpoints:


**ספרים:**

יצירת ספר -> POST /books <br>
רשימת ספרים -> GET /books<br>
ספר לפי id -> GET /books/{id} <br>
עדכון פרטי ספר -> PUT /books/{id}<br>
עדכון החזרה -> PUT /books/{id}/return/{member_id}<br>
עדכון השאלה -> PUT /books/{id}/borrow/{member_id}<br>

**משתמשים:**

יצירת משתמש -> POST /members<br>
רשימת משתמשים -> GET /members<br>
משתמש לפי id -> GET /members/{id}<br>
עריכת פרטי משתמש -> PUT /members/{id}<br>
עדכון שהמשתמש לא פעיל -> PUT /members/{id}/deactivate<br>
עדכון שמשתמש פעיל -> PUT /members/{id}/activate<br>
הוספת 1 לסכום השאלות של משתמש -> PUT /books/{id}/borrow/{member_id}<br>

**דוחות:**

משתמש עם מקסימום השאלות -> GET /reports/top-member<br>
סכום משתמשים פעילים/<br>
סכום ספרים/<br>
סכום ספרים זמינים/<br>
סכום ספרים לא זמינים -> GET /reports/summary<br>
סכום ספרים לפי זאנר -> GET /reports/books-by-genre<br>
סכום ספרים שמשתמש מחזיק כרגע -> GET /books/{id}/borrow/{member_id}<br>

### זרימת המערכת:

ברגע שמריצים את השרת נוצרות 2 הטבלאות.

המשתמש יוצר משתמש חדש,
המערכת בודקת אם האימייל ייחודי.

המשתמש יוצר ספר,
המערכת בודקת אם הז'אנר חוקי.

המשתמש מבקש לשאול ספר ->
המערכת בודקת האם -
הספר קיים וזמין,
המשתמש קיים ופעיל,
מושאל למשתמש פחות מ3 ספרים כרגע.

במידה וכל הבדיקות הצליחו:
הסטטוס של הספר משתנה ללא-זמין,
מתעדכן בפרטי הספר שהוא מושאל על ידי המשתמש,
מונה ההשאלות עולה ב1.

המשתמש מבקש להחזיר ספר ->
המערכת בודקת האם -
הספר קיים ומושאל לאותו משתמש,
המשתמש קיים ופעיל.

במידה וכל הבדיקות הצליחו:
הסטטוס של הספר משתנה לזמין,
מתעדכן בפרטי הספר שהוא לא מושאל כרגע.

**הקמת והרצת שרת:**

python -m venv venv
.venv\Scripts/activate # הקמת סביבה מבודדת
pip install 'fastapi[standard]' # הורדת הספריות הנצרכות לשרת
pip freeze > requirements.txt # הכנסת ההורדות לקובץ

uvicorn main:app --reload # הרצת הקובץ שבו השרת
http://127.0.0.1:8000 # קישור גישה להרצת הבקשות מהשרת
